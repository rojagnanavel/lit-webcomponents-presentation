What is a Component?

Web Components are a set of web platform APIs that allow developers to create custom, reusable, and encapsulated HTML tags.

Web Components consist of three main technologies:

  • Custom Elements
  • Shadow DOM
  • HTML Templates

Advantages of Web Components:

  • Encapsulation
  • Framework-Agnostic
  • Reusability
  • Native Browser Support 
							
								// user-profile.js

								// Define a custom element named 'user-profile'
								class UserProfile extends HTMLElement {
								// Constructor to initialize the component
								constructor() {
									super(); // Call the superclass constructor
									this.attachShadow({ mode: 'open' }); // Attach a shadow DOM
								}

								// Called when the element is added to the DOM
								connectedCallback() {
									this.render();
								}

								// Render the HTML content for the component
								render() {
									// Get properties passed to the component
									const name = this.getAttribute('name') || 'Guest';
									const bio = this.getAttribute('bio') || 'No bio available.';

									// Define the HTML template for the component
									this.shadowRoot.innerHTML = `
									
									

										
${name}

										
${bio}

									

									`;
								}
								}

								// Define the custom element
								customElements.define('user-profile', UserProfile);

Why Do We Need to Choose Lit?

  • Lightweight and Efficient
  • Easy to Learn and Use
  • Framework Agnostic
  • Built-in Features
  • Scalable and Reusable Components

History of Lit

  • Origins - Lit was developed by Google
  • Purpose - The main goal behind Lit’s creation was to simplify the process of building lightweight, reusable, and high-performance Web Components using web standards like HTML, CSS, and JavaScript
  • Focus on Performance - Lit was designed to offer a minimal library that focuses on fast rendering and small file sizes, unlike larger frameworks like React or Angular. It makes the most of modern browser capabilities to ensure efficient performance.
  • Evolution: In 2021, the library was rebranded as "Lit," unifying its tools and focusing on performance, ease of use, and small bundle sizes.
  • Community Growth - Lit was developed and is maintained by Google, but it has since gained an active open-source community. It continues to evolve with contributions from developers worldwide.
  • Current Status - Lit has gained popularity in design systems and microfrontends, offering a framework-agnostic approach for building shareable UI components.
